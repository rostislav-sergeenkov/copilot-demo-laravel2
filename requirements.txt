Expense Tracker — Project Description.


Goal
Record and view daily financial transactions.


Planning and tracking
The project implementation lifecycle is tracked using GitHub Issues.


Features
1. Graphical CRUD user interface for expenses.
2. Display a table of daily expenses.
3. Display a table of monthly expenses.
4. Filter expenses by category.


Technical Requirements
1. The application is built using Laravel framework and follows established coding standards and best practices for Laravel development.
2. The application source code is stored on GitHub.com.
3. Data is saved in an SQLite table. Fields:
3.1. description (string): Short description (e.g., “Lunch”).
3.2. amount (decimal): Expense amount.
3.3. category (string): Category (Groceries, Transport, Housing and Utilities, Restaurants and Cafes, Health and Medicine, Clothing & Footwear, Entertainment).
3.4. date (date): The date when the expense occurred.
4. Unit and Integrations tests are implemented on the frontend and the backend to ensure code correctness and facilitate early bug detection.
5. The application is built using GitHub Actions CI/CD pipeline, which runs tests on every pull request to the repository. It blocks merging if tests fail.
