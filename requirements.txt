Expense Tracker — Project Description.


Goal
Record and view daily financial transactions.


Planning and tracking
The project implementation lifecycle is tracked using GitHub Issues and Github Project capability.


Features
1. Graphical CRUD user interface for expenses.
2. Display a table of daily expenses.
3. Display a table of monthly expenses.
4. Filter expenses by category.
5. Export monthly expenses to an Excel-compatible CSV file.


Technical Requirements
1. The application is built using Laravel framework and follows established coding standards and best practices for Laravel development.
2. Code quality is enforced by utilizing Larastan. 
3. Unit and Integrations tests are implemented on the frontend and the backend to ensure code correctness and facilitate early bug detection.
4. The application source code is stored on GitHub.com.
5. The application is built using GitHub Actions CI/CD pipeline, which runs Larastan code quality check, Unit and Integration tests on every pull request to the repository. It blocks merging if tests fail.
6. Data is saved in an SQLite table. Fields:
6.1. description (string): Short description (e.g., “Lunch”).
6.2. amount (decimal): Expense amount.
6.3. category (string): Category (Groceries, Transport, Housing and Utilities, Restaurants and Cafes, Health and Medicine, Clothing & Footwear, Entertainment).
6.4. date (date): The date when the expense occurred.
